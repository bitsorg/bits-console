# bits Console — Configuration Reference

Quick-reference card for all configuration files, secrets, and runner conventions.
Full documentation and setup guide: [README.md](README.md)

---

## ui-config.yaml

Stored in the repository root.  **All values are public** — never put secrets here.

| Field | Default | Description |
|-------|---------|-------------|
| `title` | `bits Console` | Page title shown in the header |
| `subtitle` | — | Subtitle shown beneath the title |
| `cvmfs_repo` | — | CVMFS repository name, e.g. `sft.cern.ch` |
| `cvmfs_prefix` | — | Absolute root path on CVMFS, e.g. `/cvmfs/sft.cern.ch/lcg/releases` |
| `platforms` | `[]` | YAML list of build platform labels offered in the build dialog |
| `publish_workflow` | `cvmfs-publish.yml` | Filename of the build+publish workflow |
| `recipe_repos` | `[]` | Initial recipe repositories; overridden by provider selection in Settings |
| `spool_host` | — | Hostname or IP of the ingestion spool server (non-secret) |
| `spool_user` | `bits` | SSH user on the spool server |
| `spool_path` | — | Absolute spool root path on the ingestion server |
| `ssh_key_file` | — | SSH private key path *on the runner*; leave blank to use `SPOOL_SSH_KEY` secret |
| `runner_group` | — | Extra runner label for experiment-specific routing, e.g. `cern` |
| `config_repo` | — | Recipe config repository (`owner/repo`); overrides the `BITS_CONFIG_REPO` variable |

**Minimal example**

```yaml
title: LCG Software Console
subtitle: CERN EP-SFT
cvmfs_repo: sft.cern.ch
cvmfs_prefix: /cvmfs/sft.cern.ch/lcg/releases
platforms:
  - x86_64-el9
  - x86_64-el8
  - aarch64-el9
```

**Full example**

```yaml
title: LCG Software Console
subtitle: CERN EP-SFT
cvmfs_repo: sft.cern.ch
cvmfs_prefix: /cvmfs/sft.cern.ch/lcg/releases
platforms:
  - x86_64-el9
  - x86_64-el8
  - aarch64-el9
publish_workflow: cvmfs-publish.yml
recipe_repos: []
spool_host: ingestion.cern.ch
spool_user: bits
spool_path: /var/spool/bits
ssh_key_file: /etc/bits/spool_key
runner_group: cern
config_repo: bitsorg/lcg.bits
```

## bits.rc

Generated by the console and passed base64-encoded as the `bits_rc` workflow input.
Decoded to `~/.bits.rc` on the **build runner** before the build step runs.

| Variable | Source | Description |
|----------|--------|-------------|
| `BITS_CVMFS_REPO` | `cvmfs_repo` | CVMFS repository name |
| `BITS_CVMFS_PREFIX` | `cvmfs_prefix` | CVMFS installation prefix |
| `BITS_SPOOL_HOST` | `spool_host` | Ingestion host hostname |
| `BITS_SPOOL_USER` | `spool_user` | SSH user on the ingestion host |
| `BITS_SPOOL_PATH` | `spool_path` | Spool root path on the ingestion host |
| `BITS_SSH_KEY_FILE` | `ssh_key_file` | SSH private key path on the runner |
| `BITS_CONFIG_REPO` | `config_repo` | Recipe config repository |

## GitHub Actions secrets and variables

Set in **Settings → Secrets and variables → Actions** inside the bits-console repository.

### Secrets

| Name | Description |
|------|-------------|
| `SPOOL_SSH_KEY` | SSH private key (PEM) for the build runner to rsync to the ingestion host |
| `SPOOL_USER` | SSH username on the ingestion host (fallback when not in `bits.rc`) |
| `SPOOL_HOST` | Hostname of the ingestion host (fallback when not in `bits.rc`) |
| `SPOOL_PATH` | Spool root path on the ingestion host (fallback when not in `bits.rc`) |
| `CVMFS_REPO` | CVMFS repository name (fallback for ingest/publish runners) |

### Variables

| Name | Default | Description |
|------|---------|-------------|
| `BITS_CONFIG_REPO` | — | Default recipe config repository (`owner/repo`) |
| `CVMFS_BACKEND_TYPE` | `local` | Object store type: `local` or `s3` |
| `CVMFS_BACKEND_PATH` | — | Object store path or S3 bucket URL |
| `INGEST_CONCURRENCY` | `0` (auto) | Worker thread count for cvmfs-ingest |

## Runner labels

Register runners with these labels in **Settings → Actions → Runners**.

| Runner | Purpose | Required labels |
|--------|---------|----------------|
| Build | Compiles packages with the `bits` CLI | `self-hosted`, `bits-build-<platform>` |
| Ingest | Runs `cvmfs-ingest`; manages spool and object store | `self-hosted`, `bits-ingest` |
| Publisher | Opens CVMFS catalog transactions on stratum-0 | `self-hosted`, `bits-cvmfs-publisher` |

Platform label examples: `bits-build-x86_64-el9`, `bits-build-x86_64-el8`,
`bits-build-aarch64-el9`.

When `runner_group` is set in `ui-config.yaml`, that label is appended to every
`runs-on` selector, allowing multiple experiments to share a single
bits-console instance while routing to their own runners.

## Provider search order and colour palette

When the same package name appears in multiple active providers, the provider
with the **lowest priority number** (topmost in the list) wins.  Reorder with
the ↑ ↓ buttons in **Settings → Providers → Search order**.

Providers are assigned colours in priority order:

| Priority | Colour swatch | Use |
|----------|--------------|-----|
| 1 | 🔵 Blue | First / primary provider |
| 2 | 🟡 Amber | Second provider |
| 3 | 🟢 Green | Third provider |
| 4 | 🩷 Pink | Fourth provider |
| 5 | 🟣 Purple | Fifth provider |
| 6 | 🟠 Orange | Sixth provider |
| 7 | 🩵 Cyan | Seventh provider |
| 8 | 🟩 Lime | Eighth provider |

The same colour appears in the package list badge, the detail panel header, and
as the node fill colour in the SVG dependency graph.

**Dependency graph node legend**

| Border / fill | Meaning |
|---------------|---------|
| Provider colour fill, solid border | Normal package resolved from that provider |
| Grey fill, dashed border | Package not found in any active provider |
| Red fill | Circular dependency detected |
| Amber fill | Depth limit reached (default: 10 levels) |

## Personal Access Token (PAT)

| Scope | Required for |
|-------|-------------|
| `repo` | Reading `ui-config.yaml`, recipe files, `cvmfs-status.json` |
| `workflow` | Triggering and cancelling GitHub Actions runs |

Token is stored in the browser only (sessionStorage, or localStorage when
"Remember me" is enabled).  It is never sent to any server other than
`api.github.com`.
