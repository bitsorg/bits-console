ui-config.yaml

Workflow secrets

In agnostic runner mode (recommended) the values below are supplied via bits-console.rc from ui-config.yaml and the SSH key is stored on the runner machine. GitHub secrets are only needed as a fallback for legacy single-repo runner setups.

Configure in Settings → Secrets and variables → Actions in the repository.
SPOOL_SSH_KEY SSH private key for rsync to the ingestion host. Not needed when ssh_key_file is set in ui-config.yaml.
SPOOL_USER SSH username on the ingestion host. Not needed when spool_user is set in ui-config.yaml.
SPOOL_HOST Ingestion host address. Not needed when spool_host is set in ui-config.yaml.
SPOOL_PATH Absolute spool root path on the ingestion host. Not needed when spool_path is set in ui-config.yaml.
CVMFS_REPO CVMFS repository name (e.g. sft.cern.ch). Not needed when cvmfs_repo is set in ui-config.yaml.

Runner administration

Update the bits CLI on a self-hosted runner by dispatching the update-bits.yml workflow. Select the runner type, optionally pin a specific bits version, and click Update bits.

Config persistence

By default settings (token, repository, selected providers) are remembered only for the current browser tab and cleared when the tab is closed. Enable Remember me to persist them in localStorage across browser sessions, or use the export/import buttons to save and restore a snapshot.

Connect to GitHub

A GitHub Personal Access Token with repo and workflow scopes is required to browse packages, trigger builds, and view logs. The token is stored in sessionStorage only and cleared when you close the browser tab.
